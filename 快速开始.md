# 快速开始：从终端启动到上线

本文带你用 WZ AI 全栈工作流，从一份 PRD + 设计稿链接，一路跑到部署上线。
全程在 **Claude Code** 里用斜杠命令（`/wz:*`），不是在系统终端敲。

---

## 0. 环境准备（一次性）

| 工具 | 用途 | 检查命令 |
|---|---|---|
| Node.js ≥ 20 | 前端/后端/CDK 运行 | `node -v` |
| Claude Code | 跑 `/wz:*` 斜杠命令 | `claude --version` |
| AWS CLI + 凭证 | 部署到 Lambda（仅 deploy 阶段需要） | `aws sts get-caller-identity` |
| AWS CDK | 基础设施部署 | `npx cdk --version` |

> 没装 Claude Code：见 https://docs.claude.com（搜 Claude Code 安装）。AWS 相关只在 `/wz:deploy` 阶段才需要，前面阶段不配也能跑。

---

## 1. 放好工作流文件与 PRD

项目根目录（如 `project2/`）应包含：

```
project2/
├── .claude/          # 工作流命令、agents、skills（已复制）
├── .wz/              # 规则、模板、配置（已复制）
├── CLAUDE.md         # 入口（已复制）
├── AGENTS.md         # 入口（已复制）
└── docs/             # ← 你手动建，把 PRD 放进来
    └── 需求.docx     # ← 你的 PRD 源文件
```

- `docs/` 需要你手动建并放入 PRD（这是**输入**）。
- `docs/specs`、`docs/contracts`、`docs/evidence` 等**产物**目录由 `/wz:init` 自动创建，不用你建。
- 设计稿链接先留着，第 3 步喂给 design 阶段。

---

## 2. 从终端启动 Claude Code

```bash
cd path/to/project2
claude
```

启动后在对话框输入 `/`，应能看到 `wz:init`、`wz:prd`、`wz:design` … 一串命令。
**看得到 = 文件识别成功**；看不到说明 `.claude/commands/` 没在项目根，或用的不是 Claude Code。

---

## 3. 按阶段推进（每步都有 Gate，过了才进下一步）

### G0 ｜ `/wz:init`
检测/初始化 git、分析项目、建好全部目录、生成入口、出环境报告。
> 不是 git 仓库时会问你要不要 `git init`；缺 AWS 凭证只会提示（部署前补上即可）。

### G1 ｜ `/wz:prd docs/需求.docx`
解析 PRD → 生成 `docs/specs/product.md`、`acceptance.md`，给需求编 ID，列出**待你确认**的关键问题（存储/权限/计费/隐私）。
> 逐条回答待确认问题后再继续。

### G2 ｜ `/wz:design`
产出：UI 设计（页面×断点×状态矩阵 + design tokens）、架构、**冻结的 OpenAPI 契约**，以及一个可还原的**设计源**。

- **有设计稿** → 喂链接：
  ```
  设计稿链接：<你的链接>
  请据此用 React + Tailwind 还原 UI：注意组件结构与层次、用 Tailwind 实用类保证一致性、
  对齐细节与交互效果、动态效果用 React 状态实现。
  ```
  Figma + 装了 Figma MCP 会自动拉取；否则按截图/描述还原。
- **没有设计稿** → 不用管，`/wz:design` 会自动调 `design-prototype` skill 从 PRD 生成 HTML+CSS 设计稿原型（写到 `docs/specs/features/{N}/design/`），当作还原源和视觉回归基线。想跳过生成、让 AI 自由设计可加 `--no-prototype`。

### G3 ｜ `/wz:plan`
切 feature、拆任务（每个 ≤30min）、生成 `PLAN.md` 与各 feature 的 `tasks.md`。

### G4 ｜ `/wz:ai`
自动开发：逐任务跑「实现 → 双 AI 审查 → 标完成 → QA → 清理上下文」循环，前后端并行。
> 中途遇业务歧义/破坏性变更会暂停问你；纯技术选型自己定。

### G5 ｜ `/wz:verify`
独立验证：功能测试、视觉回归（BackstopJS 逐断点比对）、接口对照契约、安全冒烟。
> 不过 → 自动回 `/wz:ai` 修。

### G6 ｜ `/wz:review`
三类代码审查 + 安全审计，第二 AI 可在 `.wz/config.json` 切 Claude/Codex/GPT。
> 剩 P0/P1 问题需修复或人工裁决才放行。

### G7 ｜ `/wz:deploy --env preview`
后端打包上 Lambda、前端上 S3，跑冒烟。preview 全量；staging/prod 走 Canary 渐进 + 自动回滚（prod 强制人工批准）。

## 4. 部署的实际命令（在 Claude Code 引导下或手动）

```bash
# 首次：配置 AWS 凭证 + bootstrap
cd scaffold/infrastructure   # 你的真实项目里则是 infrastructure/
npm install && npx cdk bootstrap

# 看变更计划（高亮资源删除/权限扩大）
npx cdk diff Wz-preview -c env=preview

# 部署
npm run deploy:preview       # 或 deploy:staging / deploy:prod

# 冒烟（用 cdk 输出的 ApiUrl）
node ../services/api/scripts/smoke.mjs <ApiUrl>

# 前端同步到站点桶（cdk 输出 WebBucketName）
npm run build -w apps/web && aws s3 sync ../apps/web/dist s3://<WebBucketName>/
```

## 5. 常见问题

- **输入 `/` 看不到 `wz:` 命令** → 确认在项目根启动了 `claude`，且 `.claude/commands/wz/` 在根目录。
- **在系统终端直接敲 `wz:init`** → 不行，斜杠命令只在 Claude Code 对话框里用。
- **PRD 没被读到** → 确认文件在 `docs/` 下，且 `/wz:prd` 后带了正确路径。
- **deploy 报缺凭证** → `aws configure` 配好后重试；前面 G0~G6 阶段不需要 AWS。
- **想跳过某 Gate** → 不建议；Gate 是质量保证，确需豁免要人工在 `.wz/state/gate.json` 标 `waived` 并写理由。

---

## 一句话流程

```
建 docs/ 放 PRD → claude 启动 → /wz:init → /wz:prd → /wz:design(喂设计稿)
→ /wz:plan → /wz:ai → /wz:verify → /wz:review → /wz:deploy
```
