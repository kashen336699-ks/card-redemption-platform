# L0 组织规则（最高优先级）

## 安全与合规
- 所有接口默认认证；公开端点必须显式标注并经安全 Review。
- 禁止在代码、日志、提交、环境变量中出现明文密钥；密钥走 SSM / Secrets Manager。
- 涉及隐私/PII 的字段必须标注，并在 acceptance.md 列出最小化与留存策略。
- 不编写或解释恶意代码；安全 Review 只输出防御性结论。

## 分支与提交
- 每个任务独立分支或工作树：`wz/<stage>/<task-id>-<slug>`。
- 小步提交，单个提交表达单一意图；禁止千行级混合改动。
- 提交信息引用需求 ID 与任务 ID，例如 `feat(FR-002): ... [T-012]`。

## Review 与发布
- 双 AI 验证：Builder 实现 + 自测，Reviewer 独立只读审查输出 findings。
- 修复最多两轮；剩余 P1 及以上问题升级人工 Gate 裁决。
- 生产发布必须人工批准；权限扩大、资源删除、数据库变更须在部署计划中高亮。

## 不可验收措辞
禁止使用"尽量/适当/大概/优化一下"等模糊词。需求一律用 MUST / SHOULD / MAY。
